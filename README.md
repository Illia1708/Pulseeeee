# 🚀 Pulse — Соціальна мережа реального часу

## Що було реалізовано

Ви тепер маєте **повнофункціональну соціальну мережу з підтримкою реального часу**! Система синхронізується миттєво на всіх пристроях.

### ✨ Основні функції

- **🔐 Автентифікація** — реєстрація, вхід, демо-акаунт
- **📝 Пости** —創宮創, ідеї, запитання, прогрес з мудо і тегами
- **💬 Чати** — приватні діалоги з повідомленнями
- **❤️ Лайки й коментарі** — взаємодія з постами
- **👥 Профілі** — публічні профілі з підписками
- **🔔 Сповіщення** — активність в реальному часі
- **📱 Адаптивний дизайн** — працює на всіх пристроях

### 🔄 Реальні оновлення (Real-Time Sync)

Система використовує **WebSocket** для миттєвої синхронізації:

- **Пости** — з'являються на всіх пристроях одночасно
- **Лайки й коментарі** — оновлюються в реальному часі
- **Повідомлення в чатах** — синхронізуються миттєво
- **Статус користувачів** — показує, хто онлайн
- **Офлайн-режим** — все працює і без інтернету (синхронізується потім)

## 🛠️ Технічний стек

### Frontend
- **HTML5, CSS3, JavaScript** (ванільний код, без фреймворків)
- **Socket.ioClient** — WebSocket з fallback на polling
- **LocalStorage** — локальне зберігання при відсутності сервера

### Backend
- **Node.js + Express.js** — REST API
- **Socket.io** — реальний час синхронізація
- **JSON файл (db.json)** — база даних (простість та швидкість)

## 🚀 Як запустити

### 1️⃣ Встановіть залежності

```bash
cd backend
npm install
```

### 2️⃣ Запустіть бекенд (реальний час)

```bash
node backend/server.js
```

Вихід:
```
✨ Pulse backend running on http://localhost:3000
🚀 WebSocket ready for real-time updates
📁 Database: backend/db.json
```

### 3️⃣ Запустіть фронтенд

В іншому терміналі:

```bash
npm start
```

Або вручну:

```bash
python3 -m http.server 4173
```

Відкрийте **http://localhost:4173** в браузері.

## 🔌 REST API Endpoints

### Аутентифікація

```
POST /api/login          — вхід в систему
POST /api/register       — реєстрація
```

### Пости

```
POST /api/posts          — створити пост
POST /api/posts/:id/likes          — лайк/дизлайк
POST /api/posts/:id/comments       — додати коментар
```

### Чати

```
POST /api/messages       — відправити повідомлення
```

### Користувачі

```
POST /api/follow         — підписатися/відписатися
PUT /api/profile         — оновити профіль
```

### Стан

```
GET /api/state           — отримати всі дані
GET /api/health          — перевірити доступність
```

## 💡 Як це працює

### 1. **Локальна синхронізація** (без бекенду)
- Усі дані зберігаються в `localStorage`
- Додаток працює повністю офлайн

### 2. **Бекенд-синхронізація** (коли сервер запущено)
- При запуску, app завантажує стан з `/api/state`
- Всі дії (пост, лайк, коментар) відправляються на сервер
- Сервер відправляє оновлення всім клієнтам через WebSocket

### 3. **Реальний час на всіх пристроях**
- Коли інший користувач додає пост, він з'являється у вас через WebSocket
- Немає затримок, все синхронізується миттєво
- Якщо офлайн, розсинхронізація зберігається локально

## 🛑 Проблеми та розв'язання

### WebSocket не підключається?
- Переконайтесь, що `node backend/server.js` запущено
- Перевірте, що порт 3000 вільний
- Очистіть браузер кеш (Ctrl+Shift+Delete)

### Пості не синхронізуються?
- Перевірте консоль браузера (F12)
- Переконайтесь, що обидва браузери в одній мережі
- Спробуйте оновити сторінку (F5)

### База даних пуста?
- Видаліть `backend/db.json` — він перестворюється з seed-даними
- Перезапустіть `node backend/server.js`
